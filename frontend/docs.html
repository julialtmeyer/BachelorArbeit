Rest-Schnittstelle - SmartCityCozmoSteuerung

How to use

$.ajax({
	url: "http://localhost:8080/robots"
}).then(function(data) {
	console.log(data)
});
$.ajax({
	type: "POST",
	url: "http://localhost:8080/jobs",
	data: '{"robotID":"anki-01"}',
	contentType: "application/json; charset=utf-8",
	dataType: "json"
}).then(function(data) {
	console.log(data)
});

Roboter

Gibt alle registrierten Roboter mit dessen Information zurück

/robots

Response Example

{
   "robots":[
      {
         "robotID":"anki-01",
         "robotName":"Anki 1",
         "posX":1103.0,
         "posY":485.0,
         "orientation":270.0,
         "battery":4.17,
         "charger":true
      }
   ]
}
  • robots List of Robots
  • robotID String
  • robotName String
  • posX Double
  • posY Double
  • orientation Integer (0 - 259)
  • battery Double
  • charger Boolean

Image

Liefert ein Foto der Kamera eines gewünschten Ankis zurück

/image

Request Example

{
    "robotID": "anki-01"
}
  • robotID String

Response Example

{
    "robotID": "anki-01",
    "image": "data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEAYABgAAD ... tZNHKTzH//2Q==",
    "image-vision": "data:image/jpeg;base64,/1j/42AAQQZJRgABAQEAYAASDAF ... tZNHKTzH//3G=="
}
  • robotID String
  • image String (base64 encoded image)
  • image-vision String (base64 encoded image)

Drive

Lässt einen Anki an eine bestimmte Stelle auf der Karte fahren.

/drive

Request Example

{
    "robotID": "anki-01",
    "posX": 51.5,
    "posY": 14.42
}
  • robotID String
  • posX Double
  • posY Double

Adjustment

Stellt verschiedene Parameter beim Roboter ein

/adjustment

Request Example

{
    "robotID":"anki-01",
    "bitmapFace": "",
    "headPosition": 1,
    "forkPosition": 0}
}
  • robotID String
  • bitmapFace String (base64 encoded Image)
  • headPosition Double -25 <-> 44.5
  • forkPosition Double 0.0 <-> 1.0

Instruction

/instruction

Request Example

{
    "robotID": "anki-01",
    "code": "PLAY"
}
  • robotID String
  • code String ("PLAY", "STOP", "PAUSE")

Instruction

data/Gruppe3/instruction

Befehle an Cozmo senden um den Ablauf zu Steuern

  • STOP => Stop this Anki & delete all jobs
  • PAUSE => Pause the current running job
  • PLAY => Start the current job
  • RECONNECT => Reconnect this anki (* for all)
  • RESET => Reset Robot

Example

{
  "packetID" : "ins-1",
  "publisher" : "backend",
  "robot" : {
    "robotID" : "anki-01"
  },
  "instructionCode" : "PAUSE"
}

Information

data/Gruppe3/information

Robot => Backend

Überträgt Informationen über den anki an das backend

Activities:

  • image => base 64 webcam picture
  • image-vision => base 64 webcam picture
  • battery => float: The current battery voltage (not linear, but < 3.5 is low)
  • settings => info welche publish Einstellung wurde geändert
  • charger => Robot is on charger True/False

example

{
  "packetID" : "2357",
  "robotID" : "anki-01",
  "activity" : "charger",
  "timestamp" : "1599138742.285905",
  "data" : "True"
}

Log

data/Gruppe3/log

Gibt vom Roboter aus, Informationen über die ausgeführten Kommandos/Befehle des Anki an das Backend

  • packetID => Unique Identifier der logs des Roboters
  • robotID => Robot Identifier
  • jobID => Unique Identifier
  • commandID => Unique Identifier
    • nur verfügbar in einem Command log|bei einem job log ist commandID = -1
  • activity
    • job :wird verwendet wenn ein Job gestartet wird
    • drive/nop/adjustment :wird verwendet um den ausgeführten command typ anzugeben
  • state
    • job => RECEIVED/RUNNING/COMPLETE/REMOVED :Gibt den status des Jobs an
      • command => RUNNING/COMPLETE ::Gibt den status des commands an
  • timestamp => Veröffentlichkeits Zeitpunkt
  • gyrolist => Wenn es in "drive" Befehl, mit dem Status COMPLETE ist, kann eine Gyroskopdatenliste übergeben werden 
{
  "packetID" : "3422",
  "jobID" : "job-2",
  "robotID" : "anki-01",
  "activity" : "job",
  "state" : "COMPLETE",
  "commandID" : "-1",
  "timestamp" : "1599139038.416228",
  "gyrolist" : [ ]
}

Registration

data/Gruppe3/registration

Roboter anmelden (robotID muss frei sein, es wird noch keine ID ausgehandelt)

{
    "robotID" : "anki-01",
    "robotName" : "Anki 1",
    "state" : "connect"
}

Getting Started

Android

  1. Android Gerät in Debug Modus versetzen
  2. Anki Cozmo App herunterladen
  3. Anki einschalten und Mobiles Endgerät mit COZMO WLAN verbinden

Backend Starten
java -jar target\mqtt-backend-server-0.0.1-SNAPSHOT.jar

Anki Starten
python src/anki/src/main.py

Frontend
http://localhost:8080

Documentation
http://localhost:8080/docs

Anki Simulator
Wenn beim start des Anki Programmes kein USB Debugging Gerät verfügbar ist, startet eine Simulation des Roboters.
